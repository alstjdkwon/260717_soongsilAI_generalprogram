# 교직원 교육관리 도구 — 기획 문서 (v1)
> 작성일: 2026-07-12 · 용도: 클로드코드 개발 사양서 겸 맥락 인수인계
> 대상 사용자: 총무·인사팀 전세용 님 (교육훈련 담당) — **본인 전용 내부 관리 도구**
> 공모전: 숭실대 'AI를 활용한 교육·연구·행정 업무 개선 사례'

---

## 0. 한 줄 정의
그룹웨어에서 받은 자율교육 관련 PDF를 던져 넣으면, AI가 내용을 추출하고 각 신청 건의 상태를 자동 추적하며, 세영 님이 처리할 것만 골라서 보여주는 도구. **엑셀을 대체하는 게 아니라, 엑셀에 채워 넣는 일과 엑셀을 들여다보는 일을 대신한다.**

## 1. 왜 이걸 만드는가 (인터뷰 근거)
- 세영 님은 자율교육 이수현황을 그룹웨어 공문 → **엑셀 수기 입력**으로 관리 중. 본인이 "부정확하고 오류가 있을 수밖에 없다" "휴먼 에러가 발생한다"고 인정.
- 프로세스가 끊겨 있어(신청 → 교육수강 대기 → 이수보고 → 환급) 각 건의 상태를 **기억+엑셀로 추적**하는 것이 핵심 고통. 세영 님 표현: "결과 보고 했다 안 했다 이런 것들을 기억을 해뒀다가."
- 세영 님 본인이 "이거를 프로그램을 관리할 수 있으면" 좋겠다고 직접 언급. → 우리가 만드는 건 세영 님이 말한 그 '프로그램'.
- **접근성 확인됨**: 교육 업무는 SAP 안 씀. 그룹웨어 PDF 다운로드 + 엑셀이 전부 → 학생 프로토타입 가능 영역.

## 2. 설계 원칙 (스코프 이탈 방지)
1. **기존 프로세스·규정을 바꾸지 않는다.** 상태가 스스로 흐르게 만들 뿐, 단계를 없애거나 순서를 바꾸지 않는다.
2. **그룹웨어를 대체하지도, 통신하지도 않는다.** 그룹웨어 ↔ 도구 사이 이동은 '파일 던지기'와 '텍스트 복사' 두 동작으로만. (그룹웨어 API 연동은 전산팀·보안 승인 필요 → 스코프 밖, 향후 확장으로만 표기)
3. **AI는 제시, 사람은 결정.** 모든 AI 출력에 신뢰도 표기. 최종 판단(승인/반려/점수)은 항상 세영 님.
4. **자율교육에 집중.** 일반교육·인사팀주관교육은 확장 훅만 심고 지금 구현하지 않는다.

## 3. 상태 모델
```
[심사 대기] ──승인──> [이수 대기] ──서류도착──> [환급 대기] ──환급──> [완료]
     │
     └──반려──> [반려] ──재신청──> (새 건으로 [심사 대기] 진입, 이전 반려 건과 연결)
```
- **이수 대기**: 가장 오래 머무는 구간. 신청자가 교육 듣는 중. N주 경과 시 경과 알림.
- **반려**: 반려 사유 필수 입력. 재신청 시 이전 건 연결해 "지난번 왜 반려됐는지" 표시.

## 4. 단계별 처리 흐름 (사람 vs AI)

| 단계 | AI 역할 | 세영 님 역할 |
|---|---|---|
| ① 신청서 접수 | 신청서 PDF에서 이름·부서·교육명·예상비용 추출 (고정양식 → 규칙기반 파싱, 추출 정확도 표기) | PDF 업로드 |
| ② 직무 부합 검토 | 조직도 매칭으로 담당업무 표시 + 직무-교육 부합 **근거 문장** 제시 (상식추론 기반, 신뢰도 표기). 잔여 포인트 자동 계산·표시 | AI 근거 검토, 동의/수정 (교정은 축적됨) |
| ③ 승인/반려 | — | 직접 결정. 반려 시 사유 필수 입력 |
| ④ 이수 대기 | N주 경과 건 자동 표시 | 대기 |
| ⑤ 이수증 제출 | 어느 신청 건인지 자동 매칭 + OCR로 이름·교육명·수료시간·금액 추출 | 이수증/결과보고서 PDF 업로드 |
| ⑥ 환급 검토 | 신청내용 ↔ 제출서류 **대조**, 불일치 항목 하이라이트 (교육명·금액·이름) | 초록불이면 승인, 불일치만 확인 후 최종 환급/반려 |

**콜드스타트 해법**: 과거 실데이터를 받지 않는다(보안·개인정보). ②의 직무 부합 근거는 AI 상식추론으로 초안 생성 → 세영 님 교정이 쌓이며 학교 맥락에 맞춰짐.

## 5. 화면 구성

### 층 0 — 업로드 드롭존 (메인 상단 상시 노출)
- "PDF를 끌어다 놓으세요" — 신청서/이수증 **구분 없이** 여러 파일 한 번에 투입
- AI가 문서 종류 자동 판별 → 신청서면 새 건 생성, 이수증이면 기존 건에 자동 매칭
- 세영 님이 문서 종류를 고르는 클릭조차 없앤다 (배치 처리로 그룹웨어 왕복 최소화)

### 층 1 — 메인: 할일 큐 (기본 화면)
좌→우 배치 (사용 빈도순):
1. **처리 대기** — 새 신청서 N건, 환급 처리 대기 N건
2. **검토 필요** — AI 신뢰도 낮음 or 대조 불일치. 빈도는 낮지만 **시각적 무게는 가장 강하게**(빨간 배지) — 에러가 숨는 곳
3. **경과 알림** — 이수 대기 N주 초과

- 정렬: "저신뢰 + 방치 임박"이 상단
- 카드 1줄 표시 항목: **신청자명 / 부서명 / 교육명 / 현재 상태 / 경과일 / AI 플래그(색점)**
  - (색점 등 시각 요소는 추후 UI 디벨롭)

### 층 2 — 상세: 검토 화면 (카드 클릭 시)
- **왼쪽**: 원본 PDF (신청서 or 이수증)
- **오른쪽**: AI 추출 필드 (이름·부서·교육명·금액·수료시간) + 필드별 신뢰도 색상(고/중/저). 저신뢰 필드는 원본 대조해 수정 가능
- **상단 배너**: 주의 사유 한 문장 (예: "영수증 85,000원이 신청서 예상비용 70,000원과 불일치")
- **AI 근거문**: 직무 부합 추정 + 근거 (심사 단계일 때)
- **하단**: [승인] [반려] 버튼, 반려 시 사유 입력란(**필수**)

### 층 3 — 서브탭: 전체 현황 칸반
- [심사 대기 | 이수 대기 | 환급 대기 | 완료 | 반려] 열
- 세영 님 전체 조망용 + 심사위원 데모용

### 확장 훅 (지금 구현 X, 구조만)
- 상단 탭: [자율교육](활성) / [인사팀주관교육](준비중) / [일반교육](준비중)
- 데이터에 "교육유형" 필드 포함
- 기록 저장소를 **"사람(직원)" 중심**으로 설계 — 신청 건이 사람에 딸리는 구조. 나중에 "이 직원 총 이수시간" 집계(승진요건 판단용)로 자연 확장

## 6. KPI (도입 효과 측정 설계)
1. **수기 입력 단계 수** (건당 N회 → 0회) — 에러 발생 기회 제거
2. **건당 처리 시간** (접수→결정) — 판단 전 준비시간(담당업무 조회, 포인트 확인 등) 단축
3. **서류 도착 → 처리 완료 소요시간 + 방치·누락 건수** — 상태추적 문제 직접 지표
- 가치는 "빠르게"뿐 아니라 **"확실하게"**(승진 연계 민감 데이터의 에러·누락 제거)에 있음. 시간 수치가 소박해도 리스크 제거 스토리로 보완.
- 임팩트 = 건당 절감 × 월 건수 × 적용 가능 업무 수(확장 시)

## 7. 세영 님께 확인 대기 항목 (이메일)
- [ ] 자율교육 신청서 양식 (빈 폼)
- [ ] 결과보고서 양식 (빈 폼)
- [ ] 현재 관리 엑셀의 열 구조 (헤더만, 데이터 제외)
- [ ] 잔여 포인트 관리 방식 + 열 구조 (초기값/한도/사용내역 어디서 관리하는지)
- [ ] 담당업무 기재된 조직도/직무 자료
- [ ] 이수 기한 존재 여부 (예: 승인 후 4주)
- [ ] **승인/반려 결과를 신청자에게 어떻게 통보하는지** (그룹웨어 회신? 메일? 구두?) — 회신 필수면 회신문 초안 생성 기능 추가 검토
- [ ] (KPI baseline) 심사 소요시간 / 환급 소요시간 / 월 누락 건수(단계 안 박고 열어서) / 옮겨적기 건당 시간 / 월 신청 건수
- ※ **판단 일화 3~4개**(직무+교육→승인/반려 사유)는 프로토타입 1차 시연 때 요청 (few-shot 예시용)
- ※ 실데이터 샘플은 개인정보로 불가 → 양식만 받아 가상데이터로 프로토타입
- ※ 모든 연락에 "평가와 무관" 명시 (형평성 민감)

## 8. 기술 메모 (클로드코드용)
- 신청서 = 고정양식 → 규칙기반 파싱 우선 / 이수증·결과보고서 = 기관마다 폼 상이 → AI(OCR+LLM) 추출
- 브라우저 스토리지 사용 불가 환경 가능성 → 상태는 앱 내 관리 or 명시적 저장소
- 프로토타입 데이터는 명백한 가상값(홍길동, 테스트1 등)으로 채우고 "임의 생성 데이터" 명시
- 레퍼런스 패턴: Ramp/Brex (AI 이상탐지 + 플래그 이중노출), 문서처리 신뢰도 3단계(고/중/저 → 사람검토 라우팅), Zendesk (방치 경고)
