## 1. 적용한 방법론

**전체 프레임: 컨설팅 케이스 인터뷰 4단계**
`Clarification(업무범위 러프 파악) → Structuring(Where축+Why축 설계) → Analysis(인터뷰로 데이터 채움) → Solution(설계·개발)`

**부서/업무 스크리닝 — 3축**
- 반복성: 같은 형태의 일이 자주 발생하는가
- 규칙성(비정형→정형): 판단 기준이 문서화된 규칙으로 존재하는가, 인풋이 비정형이고 아웃풋이 정형인가
- 접근성: 작업이 열린 도구(엑셀·문서·메일)에서 일어나는가, 아니면 SAP 등 폐쇄 기간계 안에서 끝나는가 (학생 프로토타입 실현 가능성 결정)

**보조 평가축(케이스 확정 후 적용)**: 담당자의 고통 인식 여부, 시즌성/마감압박, 성과 측정 가능성, 데이터 확보 가능성(공개 자료만으로 데모 가능한지), 확장성.

**Why축(원인 분석) — 시간방정식 4요인** (창업지원단 케이스에서 도출, 이후 범용 렌즈로 재사용)
```
행정 총시간 = Σ(건수 × 건당시간 × 재작업배수) + 횡단관리시간
```
→ 건수과다 / 건당시간과다 / 재작업과다 / 횡단오버헤드. 어느 부서·업무가 병목으로 확정되든 이 4칸에 데이터를 채우는 방식으로 분석.

**핵심 인터뷰 원칙**
1. **열린 질문 먼저, 가설은 체크리스트로 나중에.** 가설을 먼저 제시하면 인터뷰이가 그 틀 안에서만 답해 유도신문이 됨. 순서: (1)열린 질문으로 전체 프로세스 청취 → (2)"가장 시간 드는 단계는?"으로 자발적 병목 청취 → (3)언급 안 된 가설만 후반에 확인.
2. **AI는 판단이 아니라 근거 제시.** 재량·정성 판단(직무 연관성 점수, 인사 재량 등)은 항상 사람 몫. AI는 정보 대령 + 근거 문장 제시 + 신뢰도 표기까지만.
3. **데스크 리서치는 후보를 좁히는 데까지만 신뢰.** "진짜 아픈지"는 실제 담당자만 답할 수 있음. 이 원칙이 반복 검증됨 — 창업지원단(정산·보고 반복 가설이 매니저 피드백으로 전면 기각), 학사팀 김예은(예외처리 유형은 문서화돼 있었지만 실제 발생 건수는 적음), 전세용(법정교육은 본인 소관 아님) 케이스 모두 데스크 리서치 가설이 현장에서 뒤집힘.
4. **프로세스는 바꾸지 않고 상태추적 부담만 제거.** "프로세스 파괴"가 아니라 "프로세스는 그대로, 상태가 스스로 흐르게" — 권한도 없고, 형평성에 예민한 인터뷰이에게도 안전한 방향.
5. **MECE 점검 시**: Where축은 ME(상호배제)뿐 아니라 CE(전체포괄)도 확인 — 특정 단계에 안 속하는 횡단업무 버킷이 빠지기 쉬움. Why축은 위 시간방정식처럼 항등식으로 분해해야 MECE가 보장됨(임의로 나열한 가설은 MECE가 아님).

## 2. 부서 탐색 히스토리

| 부서/담당자 | 상태 | 판단 근거 (한 줄) |
|---|---|---|
| **창업지원단 (송수진 매니저)** | 인터뷰 완료, 사실상 종료 | 데스크리서치 가설("정산·보고 반복")이 매니저 피드백으로 전면 기각 → "수주(매번 창작이라 자동화 부적합)+운영" 재조사 → 이중입력은 확인됐으나 "그냥 보고 다시 치는" 방식이라 자동화 여지 낮음, 재작업은 비정형이라 패턴 없음, 공모 발굴·다중사업관리 가설도 기각됨 |
| **학사팀 김예은 (다전공·이수구분·졸업·제증명)** | 기각 | 데스크리서치로는 최유력(졸업사정 자동화 이미 존재, 예외처리 6종 taxonomy 확인)이었으나, 민성님의 실사용 지식으로 기각 — 이수구분 처리는 연 2회 10일만 접수, 단과대 조교의 선제 필터링으로 학사팀 도달 건수 자체가 적고 민원도 적음 |
| **장학팀** | 보류 (존재 확인 필요) | 초기 데스크리서치에서 RAG 적합도 최고로 평가됐으나, 최신 기구표 확인 결과 학생처 산하에 "장학팀" 명칭이 안 보임(학생지원팀에 흡수됐을 가능성) → 재확인 안 된 채 보류 |
| **총무·인사팀 전세용 (교육훈련·비정규직 인사)** | ★ 진행 중 (현재 케이스) | 인터뷰로 검증 완료 — 교육기획은 AX 부적합(본인 판단), 이수관리(그룹웨어→엑셀 수기입력)에서 휴먼에러+상태추적 부담(방치) 확인, SAP 미사용으로 접근성 통과. 자율교육 관리도구로 스코프 확정. 다만 사후 자료요청 이메일에 5일째 무응답 상태 |

**백업 후보 (5개 부서 데스크리서치 완료, 실제 인터뷰 미실시)**: 창업지원단·전세용 인터뷰에서 파생된 확장 후보로 **김정효 님(총무·인사팀, 복리후생·경조사·행사·서무)**이 유력 — 경조사 신고(비정형)→규정 대조→정형 처리 구조가 선명하고 SAP 밖일 가능성 높음, 전세용 님과 같은 팀이라 소개받기 용이. 그 외 중앙도서관·진로취업센터·상담센터는 데스크리서치만 완료된 상태.
